Managing Configurations

By default CodeIgniter store configuration files into application/config/ directory. However, it is possible, to rename or relocate your config folder, even you can have different sets of specific configurations directories.

Renaming the Config Folder

If you would like to rename your config folder for add security or create a more manageable directory structure you may do so as long as you open your main index.php file and set its name using the $config_folder variable:

$config_folder = "config";

Relocating your Config Folder

It is possible to move your config folder to a different location on your server than your application folder. To do so open your main index.php and set a full server path in the $config_folder variable.

$config_folder = "/path/to/your/config";

Running Multiple Instances for a single CodeIgniter Application

If you would like to reuse an application installation to manage different instances but with a different configuration, simply create copy of the config directory located inside your application folder and rename it or change their path.

Note:  Each of your instances will need its own index.php file which calls the application. The index.php file can be named anything you want.

For example, let's say you want to run two instances of a warehouse control application because, for any reason their data cannot be mixed and needs to store to a diferent database.

You can avoid to write more code or create an entire copy of your application by sharing a single installation of your application. To accomplish this you could structure your application like this:

application/config/
application/controllers/
application/errors/
application/libraries/
application/models/
application/views/
config/foo/
config/bar/
index.php
bar/index.php

To select a particular configuration directory for use requires that you open the index.php file for each instance you created and set the $config_folder variable. For the preceding example you should do this:

On index.php

$config_folder = "config/foo"; On bar/index.php

$system_path = "../system";

$application_folder = "../application";

$config_folder = "../config/bar";

Now you can set a custom configuration for each instance as usual. You can still use the ENVIROMENT configuration files and folders. This is described in more detail in the environment section of the Config Class documentation.
